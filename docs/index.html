Pulse

Pulse is a small Wasm-friendly app layer over a clean runtime substrate.

It gives you a paved road for the recurring pain points every edge/Wasm app hits:

  • route and session behavior
  • local listen() vs portable bind()
  • consistent config/env/profile access
  • asset backend selection without bundling static files into the Wasm artifact
  • optional GRIP gateway support when you need proxy-managed realtime flows

The runtime stays small. The app layer smooths workflow.

TL;DR

  • pulse.runtime is the substrate. It owns PulseRuntime, Router, Context, sessions, broadcast seam, and bind().
  • pulse is the paved road. It wraps the runtime with config/profile/env helpers, router passthrough methods, and profile-scoped listen() convenience.
  • pulse.node owns host lifecycle. It provides an explicit profile-local host binder for Node-style local hosting.
  • pulse.grip is optional gateway capability. GRIP/Pushpin/Fastly Fanout mechanics stay additive.
  • pulse.assets keeps assets out of the Wasm bundle. Asset backends are middleware/config, not core runtime.

Packages

Package Purpose
pulse Opinionated app/workflow layer. Normal entry point.
pulse.runtime Runtime substrate: PulseRuntime, Router, context, sessions, bind(), broadcast seam.
pulse.node Node request/response normalization and profile-local host binder.
pulse.grip Optional GRIP gateway middleware and broadcaster for Pushpin/Fastly Fanout-style gateways.
pulse.assets Optional asset middleware for local/hosted/bucket-backed static assets outside the Wasm artifact.

Minimal app

ts
import { Pulse, defineConfig } from 'pulse';

const config = defineConfig((env) => ({
  profile: env('PULSE_PROFILE', 'local'),
  profiles: {
    local: {
      assets: { mode: 'local', dir: './public' }
    },
    edge: {
      assets: {
        mode: 'bucket',
        endpoint: env('ASSET_ENDPOINT'),
        bucket: env('ASSET_BUCKET'),
        credentials: {
          key: env('ASSET_KEY'),
          secret: env('ASSET_SECRET')
        },
        ttl: 3600,
        immutable: true
      }
    }
  }
}));

const app = new Pulse({ config });

app
  .get('/health', async (ctx) => ctx.reply({ ok: true }))
  .post('/compute/:id', async (ctx) => ctx.reply({ id: ctx.params.id, ok: true }));

export default app.bind();

Local Node host

bind() is always available. listen() is only available when the active profile defines a profile-local host binder.

ts
import { Pulse, defineConfig } from 'pulse';
import { node } from 'pulse.node';

const config = defineConfig((env) => ({
  profile: env('PULSE_PROFILE', 'local'),
  profiles: {
    local: {
      host: node({
        port: env.number('PORT', 3000),
        baseUrl: env('BASE_URL', 'http://localhost:3000')
      }),
      assets: { mode: 'local', dir: './public' }
    },
    edge: {
      assets: { mode: 'hosted', origin: env('ASSET_ORIGIN') }
    }
  }
}));

const app = new Pulse({ config });
await app.listen();

If the active profile does not define host, the app is fetch-native by default and app.bind() is the portable runtime seam.

Config access

ts
const active = app.config();
const assets = app.config('assets');

app.env('ASSET_BUCKET');
app.env.number('TTL', 3600);
app.env.boolean('DEBUG', false);

pulse.config is code so it can be explicit, commented, and Wasm-build-friendly. Environment resolution happens through the env resolver passed into defineConfig(...), not through app code reaching directly into host globals.

Runtime options

Runtime options are the only app-layer options that shallow-merge by key:

  1. Pulse runtime defaults
  2. active profile runtime
  3. new Pulse({ runtime }) overrides
ts
const app = new Pulse({
  config,
  runtime: {
    defaultHttpMode: 'stream'
  }
});

Profile host does not merge. If a profile needs Node-style local hosting, define host on that profile. If it does not, the profile is fetch-native.

Assets

ts
import { AssetManager } from 'pulse.assets';

app.use('/assets', new AssetManager(app.config('assets')));

The app layer resolves the active profile. AssetManager consumes the resolved explicit-mode shape: local, hosted, or bucket. Route code does not branch on local/prod/edge, and the middleware does not infer from profile labels.

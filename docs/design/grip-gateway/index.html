GRIP gateway

GRIP is a lower-level HTTP gateway/proxy framing protocol.

gatewayrealtime

GRIP gateway boundary

GRIP is a lower-level HTTP gateway/proxy framing protocol.

Pushpin and Fastly Fanout are implementations or provider expressions of that pattern.

Pulse core should not know GRIP headers, WebSocket-over-HTTP control messages, or provider hold semantics.

Package role

pulse.grip owns:

  • GRIP ingress verification
  • WebSocket-over-HTTP context mapping
  • hold response formatting
  • subscribe/unsubscribe/disconnect gateway helpers
  • GRIP-backed broadcaster
  • context namespace decoration

Context namespace

Gateway behavior is explicit:

ts
ctx.grip.subscribe(...)
ctx.grip.unsubscribe(...)
ctx.grip.disconnect(...)

Aliases such as ctx.fanout are allowed via ctxName, but the package/class name stays honest: GRIP gateway.

Rule

ctx.subscribe(...) should remain core behavior.

ctx.grip.subscribe(...) is gateway behavior.

Do not silently mutate core semantics because a gateway middleware was installed.

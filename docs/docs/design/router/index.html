Router boundary

Router exists to scope behavior, not to become a framework.

It owns:

  • path-scoped routing
  • middleware order
  • nested routers
  • lifecycle hooks
  • channel defaults
  • explicit error boundaries

It does not own:

  • provider capability
  • host lifecycle
  • advanced path language
  • framework adapters

Path grammar

Allowed:

  • literals
  • multiple :params
  • one trailing *
  • nested relative routers

Not included:

  • regex
  • optional params
  • mid-path wildcards
  • path generation DSL

Flow

Precedence is chain order and nesting order.

Normal handlers use (ctx, next).

Errors are handled only by .error((err, ctx, next) => ...).

Inside an error handler:

  • return Response to handle and terminate
  • call next(err) to keep bubbling
  • call next() to recover and resume normal flow

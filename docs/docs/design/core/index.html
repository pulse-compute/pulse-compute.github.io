Runtime boundary

pulse.runtime is deliberately small.

It owns:

  • PulseRuntime
  • Router
  • normalized session behavior
  • context shape
  • broadcast seam
  • wire envelopes
  • bind() contract

It does not own:

  • app/workflow profile selection
  • host lifecycle convenience
  • asset backend selection
  • GRIP/Fanout/Pushpin framing
  • provider-specific handoff
  • long-lived connection management outside the normalized session contract

App layer

pulse sits above this runtime. It may smooth workflow through:

  • config/env/profile resolution
  • app.config()
  • router forwarding
  • profile-scoped listen() convenience

That smoothing must not redefine runtime semantics.

Rule

Runtime can expose generic application-facing verbs:

  • reply
  • subscribe
  • broadcast
  • pass through
  • reject

Runtime should not expose provider protocol nouns as first-class concepts.

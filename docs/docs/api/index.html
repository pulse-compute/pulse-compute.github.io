Pulse API

Packages

Package Purpose

pulse

ts
import { Pulse, Router, defineConfig } from 'pulse';

defineConfig(input)

ts
type PulseConfigInput<T = Record<string, unknown>> =
  | T
  | ((env: PulseEnvResolver) => T);

function defineConfig<T>(input: PulseConfigInput<T>): PulseConfigInput<T>;

The config function receives a narrow env resolver.

ts
const config = defineConfig((env) => ({
  env: env.define(['PULSE_PROFILE', 'ASSET_BUCKET']),
  profile: env('PULSE_PROFILE', 'local'),
  ttl: env.number('TTL', 3600),
  debug: env.boolean('DEBUG', false),
  profiles: {
    local: { assets: { mode: 'local', dir: './public' } },
    edge: { assets: { mode: 'bucket', bucket: env('ASSET_BUCKET') } }
  }
}));

PulseEnvResolver

ts
type PulseEnvResolver = {
  (name: string): string | undefined;
  (name: string, fallback: string): string;
  define(names: string[] | Record<string, unknown>): string[] | Record<string, unknown>;
  number(name: string): number | undefined;
  number(name: string, fallback: number): number;
  boolean(name: string): boolean | undefined;
  boolean(name: string, fallback: boolean): boolean;
};

env() resolves from the environment source supplied to the app layer. App code does not directly read host globals.

new Pulse(options?)

Pulse extends the runtime substrate and adds app/workflow helpers.

ts
type PulseAppOptions<TConfig = Record<string, unknown>> = {
  config?: PulseConfigInput<TConfig>;
  env?: Record<string, string | number | boolean | null | undefined> | (() => Record<string, string | number | boolean | null | undefined>);
  runtime?: Partial<PulseOptions>;
};

app.config()

ts
app.config();          // resolved active profile config
app.config('assets');  // resolved namespace from active profile

Rules:

  • returns resolved config only
  • throws on missing namespace
  • profile selection comes from config profile, then local if present, then first profile key

app.env(...)

ts
app.env('KEY');
app.env('KEY', 'fallback');
app.env.number('TTL', 3600);
app.env.boolean('DEBUG', false);

App router helpers

Pulse owns a default internal router and forwards the small router surface:

ts
app.get(path, handler);
app.post(path, handler);
app.use(handler);
app.use(path, handler);
app.mount(path, router);
app.channel(definition);
app.on('connect', handler);
app.on('disconnect', handler);
app.error(handler);
app.bind();
app.handle(request, runtime?);

app.listen(options?)

listen() delegates to the active profile’s host binder. If the active profile defines no host binder, the app is fetch-native by default and app.bind() remains the portable runtime seam.

host is profile-local and explicit:

ts
import { node } from 'pulse.node';

const config = defineConfig((env) => ({
  profile: env('PULSE_PROFILE', 'local'),
  profiles: {
    local: {
      host: node({ port: env.number('PORT', 3000) })
    }
  }
}));

pulse.runtime

ts
import { PulseRuntime, Router } from 'pulse.runtime';

Use this when you want the lower-level substrate without the app layer.

new PulseRuntime(options?)

ts
type PulseOptions = {
  gateway?: PulseGatewayLike;
  broadcaster?: PulseBroadcasterLike;
  defaultHttpMode?: 'stream' | 'sse' | 'response' | 'long-poll';
  strictProxy?: boolean;
  channelPrefix?: string;
  verify?: 'auto' | 'required' | 'off';
  trace?: PulseTraceHooks;
  onError?: (err: unknown, ctx: { phase: string }) => void;
};

runtime.createSession(request)

Creates a normalized session.

runtime.handle(request, handler)

Runs a low-level session handler.

runtime.broadcast(channel, data, options?)

Delegates to the configured broadcaster. Throws if no broadcaster exists.

Router

Available from both pulse and pulse.runtime.

ts
const router = new Router({ pulse });

Methods

ts
router.get(path, handler);
router.post(path, handler);
router.use(handler);
router.use(path, handler);
router.mount(path, childRouter);
router.channel(definition);
router.on('connect', handler);
router.on('disconnect', handler);
router.error(handler);
router.bind();
router.handle(request, runtime?);

Path grammar

Allowed:

  • literal segments: /health
  • params: /users/:userId/settings/:setting
  • one trailing wildcard: /assets/*

No regex, optional params, or mid-path wildcards.

Provider packages may add explicit namespaces such as ctx.grip.

pulse.grip

ts
import { GripGateway } from 'pulse.grip';

GripGateway is optional middleware and gateway implementation.

ts
const grip = new GripGateway({ grip: 'http://localhost:5561', ctxName: 'grip' });
app.use(grip);

It decorates context with ctx.grip by default.

ts
ctx.grip!.subscribe(channel);
ctx.grip!.unsubscribe(channel);
ctx.grip!.disconnect(reason);
ctx.grip!.reply(payload);

GripGateway.broadcaster(options)

Provides a broadcaster for new Pulse({ broadcaster }) or app-layer new Pulse({ broadcaster }).

pulse.assets

ts
import { AssetManager, AssetBucket, createAssets } from 'pulse.assets';

AssetManager is the high-level asset middleware for resolved app/profile config. It requires an explicit mode: local, hosted, or bucket.

ts
app.use('/assets', new AssetManager(app.config('assets')));

createAssets(config) is equivalent to new AssetManager(config).

Asset config shapes

ts
type AssetsConfig =
  | { mode: 'local'; dir: string; ttl?: number | false; immutable?: boolean }
  | { mode: 'hosted'; origin: string; prefix?: string; ttl?: number | false; immutable?: boolean }
  | ({ mode: 'bucket' } & AssetBucketOptions);

Asset profile resolution belongs in the app layer. Asset middleware consumes the resolved shape and does not infer backend behavior from profile names, runtime, or env.

AssetBucket remains the lower-level S3-compatible bucket middleware for direct bucket-only usage.

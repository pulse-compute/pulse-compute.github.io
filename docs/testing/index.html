What pnpm test covers

Package-level contract tests under packages/*/test/**/*.test.ts.

These should stay focused on package behavior:

  • pulse session, router, trace, broadcast seam
  • pulse.node request/response normalization
  • pulse.grip gateway/middleware/broadcaster behavior

What pnpm test:e2e covers

The top-level test/ package contains the current cross-environment harness slice.

It covers:

  • Node listen() / bind behavior
  • compute/fetch-style Router.bind() behavior
  • routing and middleware flow
  • lifecycle hook wiring
  • broadcast seam behavior with fake broadcaster
  • error recovery flow

What is intentionally not covered yet

This slice does not start Pushpin and does not validate real GRIP/Fanout gateway behavior.

Deferred:

  • Pushpin local Docker harness
  • real GRIP handoff
  • channel broker semantics
  • gateway close/refresh behavior
  • Fastly local compute/Viceroy parity

Test posture

Prefer tests that verify Pulse-owned behavior. Do not retest Pushpin or provider internals in the core suite.

Use provider/gateway tests only to prove the seam:

  • context decoration
  • subscribe/reply handoff behavior
  • broadcaster calls
  • ws unsubscribe/disconnect behavior
